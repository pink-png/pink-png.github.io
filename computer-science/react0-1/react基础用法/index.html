# jsx 语法规则

  1. 定义虚拟 DOM 时，不要写引号。
  2. 标签中混入 JS 表达式时要用 {}。
  3. 样式的类名指定不要用 class，要用 className。
  4. 内联样式，要用 style= {{key:value}} 的形式去写。
  5. 只有一个根标签
  6. 标签必须闭合
  7. 标签首字母
    (1). 若小写字母开头，则将该标签转为 html 中同名元素，若 html 中无该标签对应的同名元素，则报错。
    (2). 若大写字母开头，react 就去渲染对应的组件，若组件没有定义，则报错。

# react 中定义组件

  1. 函数式组件
  1. 类式组件
  1. 2 种创建方式的区别

    • 没有生命周期方法：函数组件没有像类组件那样的生命周期方法（如 componentDidMount、componentDidUpdate 等），这使得处理组件的挂载、更新和卸载过程相对困难。不过，函数组件可以使用 React 的钩子函数（Hooks）来模拟类似的行为。

    • 没有内部状态管理：函数组件无法直接定义和管理内部状态（state），这意味着你无法在函数组件中使用 setState 方法来更新组件状态。但是，你可以使用 React Hooks 中的 useState 来管理局部状态。

    • 无法使用 Refs：在函数组件内部，无法直接使用 ref 属性或 React.createRef () 创建的引用。不过，React 提供了 useRef 钩子来解决这个问题。

    • 不适用于复杂逻辑：如果你的组件需要处理复杂的业务逻辑，包含多个生命周期方法和状态管理，函数组件可能会显得过于冗长和难以维护。在这种情况下，使用类组件可能更合适。

# react 组件的三大属性

# state

state 是什么？

State 是 React 组件内部的一种机制，用于存储和管理组件的可变状态。
State 是私有的，只能在组件内部进行修改。组件可以通过调用 setState 方法来更新 State 并重新渲染，React 将自动处理 UI 更新。
在类组件中，state 被定义在 constructor 方法中，并且可以通过 this.state 进行访问。

# props

props 是什么？

Props 是 React 组件传递数据的一种机制，用于从父组件向子组件传递数据。
Props 是只读的，子组件无法修改自己接收到的 props。父组件可以通过更改传递给子组件的 props 来影响子组件的行为和呈现。
在函数组件中，props 作为参数传递给组件函数。在类组件中，props 可以通过 this.props 进行访问。

# refs

Refs 是 React 提供的一种访问 DOM 元素或组件实例的方法。

Refs 主要用于以下几个方面：

访问 DOM 元素：通过 ref 属性将 DOM 元素与 React 组件关联起来，从而可以直接访问和操作该 DOM 元素。
访问组件实例：通过 ref 属性将组件实例与另一个组件关联起来，从而可以直接调用被关联组件的方法或访问其属性。
在函数组件中，可以使用 useRef 钩子来创建和访问引用。在类组件中，可以使用 createRef 方法来创建和访问引用。

# react 非受控组件 和 受控组件

在 React 中，有两种常见的表单组件模式，即受控组件和非受控组件。它们之间的区别主要在于对表单数据的处理方式。

  1. 非受控组件

非受控组件是指将表单元素的值交由 DOM 本身来管理，而非由 React 组件状态来管理。它们依赖于对 DOM 节点的引用（ref）来获取表单值。

当用户输入时，可以直接通过引用访问到表单元素的当前值。通常使用 ref.current.value 来读取或操作表单的值。

class MyForm extends React.Component {
  inputRef = React.createRef();

  handleSubmit = (event) => {
    event.preventDefault();
    console.log("Submitted value:", this.inputRef.current.value);
  }

  render() {
    return (
      <form onSubmit={this.handleSubmit}>
        <input type="text" ref={this.inputRef} />
        <button type="submit">Submit</button>
      </form>
    );
  }
}
  1. 受控组件
    受控组件是指由 React 组件来管理表单元素的状态和值。通常，通过使用 state 来存储表单元素的值，并使用 onChange 事件来更新 state。

当用户输入时，React 会更新状态并重新渲染组件，从而保持表单值与组件状态的同步。这使得可以通过 this.state 获取或操作表单的当前值

class MyForm extends React.Component {
  constructor(props) {
    super(props);
    this.state = {
      inputValue: ""
    };
  }

  handleChange = (event) => {
    this.setState({ inputValue: event.target.value });
  }

  handleSubmit = (event) => {
    event.preventDefault();
    console.log("Submitted value:", this.state.inputValue);
  }

  render() {
    return (
      <form onSubmit={this.handleSubmit}>
        <input
          type="text"
          value={this.state.inputValue}
          onChange={this.handleChange}
        />
        <button type="submit">Submit</button>
      </form>
    );
  }
}

总结：

受控组件适用于需要在 React 组件中对表单值进行处理和验证的情况。它提供了更多的控制和灵活性，但需要更多的代码和状态管理。
非受控组件适用于独立的简单表单，将表单值的控制权交给 DOM 本身。它通常比较简洁，但在某些情况下可能会更难实现复杂的表单验证和自定义处理逻辑。
