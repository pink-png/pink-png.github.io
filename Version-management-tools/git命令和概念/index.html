Git 命令和概念

Git 概念

序号 概念 描述
1 默认分支名称 通常是 main，但也有可能是 master 或其他自定义名称
2 默认远程名称 通常是 origin
3 当前分支参考 HEAD
4 HEAD 的父级 HEAD^ 或 HEAD~1
5 HEAD 的祖父母 HEAD^^ 或 HEAD~2

Git 命令

序号 命令 描述
1 git init 初始化一个新的 Git 仓库在当前目录（注意：git init <directory> 是不正确的，应该先在目标目录中使用 cd 命令，然后执行 git init）
2 git clone <url> 克隆一个远程 Git 仓库到本地
3 git add <file> 或 git add . 添加文件到暂存区（准备提交）。使用 . 代替 <file> 可添加当前目录中的所有文件
4 git commit -m "<message>" 或 git commit -a -m "<message>" 提交更改到本地仓库。-a 标志用于自动添加所有已跟踪文件的更改
5 git reset <file> 从暂存区移除文件
6 git mv <current path> <new path> 移动或重命名文件
7 git rm <file> 或 git rm --cached <file> 从存储库中删除文件或仅从暂存区中删除文件
8 git branch 显示分支列表。可配合 -a、-r、-v 标志使用
9 git branch <branch> 或 git checkout -b <branch> 创建一个新分支或切换到一个新分支并创建它
10 git checkout <branch> 切换到指定分支
11 git branch -d <branch> 或 git branch -D <branch> 删除一个分支。-D 强制删除
12 git merge <branch to merge into HEAD> 合并指定分支到当前分支
13 git rebase <branch to rebase from> 变基操作，将当前分支的提交移动到另一个分支上
14 git checkout <commit id> 切换到指定提交
15 git revert <commit id> 创建一个新的提交来撤销指定提交的更改
16 git reset <commit id> 或 git reset --hard <commit id> 重置当前分支的 HEAD 到指定提交。--hard 标志会丢弃所有未提交的更改
17 git status 显示存储库的状态
18 git log 显示提交历史
19 git diff 或 git diff --staged 显示对未暂存文件的更改或显示对暂存文件的更改
20 git diff <commit id 01> <commit id 02> 显示两次提交之间的变化
21 git stash 或 git stash save "<message>" 临时存储更改而不提交
22 git stash list 列出所有存储的更改
23 git stash apply <stash id> 或 git stash apply stash@{0} 应用一个存储的更改
24 git stash drop <stash id> 删除一个存储的更改
25 git stash clear 删除所有存储的更改
26 git stash pop <stash id> 应用并删除一个存储的更改
27 git stash show <stash id> 显示存储中的更改
28 git remote add <remote name> <url> 添加一个远程仓库
29 git remote -v 显示远程仓库的 URL，以及它们分别用于推送和拉取的别名（fetch 和 push）
30 git remote remove <remote name> 删除名为 <remote name> 的远程仓库
31 git remote rename <old name> <new name> 将远程仓库的别名从 <old name> 重命名为 <new name>
32 git fetch <remote name> 从名为 <remote name> 的远程仓库获取所有分支的最新更改，但不合并到当前分支
33 git fetch <remote name> <branch> 从名为 <remote name> 的远程仓库的 <branch> 分支获取最新更改，但不合并到当前分支
34 git pull <remote name> <branch> 从名为 <remote name> 的远程仓库的 <branch> 分支拉取更改，并自动合并到当前分支
35 git push <remote name> 将当前分支的更改推送到名为 <remote name> 的远程仓库的对应分支（默认推送当前分支）
36 git push <remote name> <branch> 将 <branch> 分支的更改推送到名为 <remote name> 的远程仓库的 <branch> 分支
