微信⼩程序有哪些重要⽂件

页面文件

  1. WXML (WeiXin Markup Language)：类似于HTML，用于描述页面的结构。
  2. WXSS (WeiXin Style Sheets)：一套样式语言，用于描述WXML组件的样式。
  3. JS (JavaScript)：逻辑处理文件，用于处理页面的交互逻辑、网络请求等。
  4. JSON：配置文件，用于小程序的全局配置、页面配置等。
    app.json：小程序的全局配置文件，包括页面注册、网络超时时间、窗口样式等。
    页面.json文件：页面的配置文件，如页面的标题、tabBar配置等。

全局配置文件

  1. app.json 必须要有这个⽂件，如果没有这个⽂件，项⽬⽆法运⾏，因为微信框架把这个作为配置⽂件⼊⼝，整个⼩程序的全局配置。包括⻚⾯注册，⽹络设置，以及⼩程序的window 背景⾊，配置导航条样式，配置默认标题。
  2. app.js 必须要有这个⽂件，没有也是会报错！但是这个⽂件创建⼀下就⾏ 什么都不需要写以后我们可以在这个⽂件中监听并处理⼩程序的⽣命周期函数、声明全局变量。
  3. app.wxss 配置全局 css

微信⼩程序怎样跟事件传值

  1. data-* 属性：在WXML中给元素添加data-前缀的属性，然后在事件处理函数中通过e.currentTarget.dataset来获取这些值。
  2. 使用bind/catch事件绑定：例如bindtap、catchtap等，可以在事件处理函数中直接访问事件对象e，从而获取事件相关的数据。
  3. 在事件处理函数中直接传递参数：通过调用事件处理函数时传递参数来实现。

⼩程序的 wxss 和 css 有哪些不⼀样的地⽅？

wxss和css大部分语法是相同的，但也有以下一些不同点：

  1. 尺寸单位：wxss中支持rpx（responsive pixel）作为尺寸单位，它可以根据屏幕宽度进行自适应。
  2. 图片引入：wxss中引入图片通常使用相对路径或项目中的图片资源，而不是直接的外链地址。
  3. 选择器：wxss支持部分CSS3选择器，但并非全部。
  4. 全局样式与局部样式：在app.wxss中定义的样式为全局样式，会影响整个小程序的所有页面；而在页面wxss文件中定义的样式仅对当前页面有效。

⼩程序关联微信公众号如何确定⽤户的唯⼀性

  1. 使用openid：用户在小程序中登录后，可以通过调用微信提供的API获取到用户的openid，这个openid对于同一微信用户在同一小程序下是唯一的。
  2. 使用unionid：如果小程序与公众号关联了同一个开放平台账号，并且用户同时关注了公众号并在小程序中登录，那么可以通过调用微信提供的API获取到用户的unionid，这个unionid对于同一微信用户在不同的小程序和公众号下都是唯一的。

使⽤ wx.getUserInfo ⽅法 withCredentials 为 true 时 可获取encryptedData ，⾥⾯有 union_id 。后端需要进⾏对称解密

微信⼩程序与vue区别

  1. 生命周期：两者的生命周期钩子有所不同，需要根据各自的文档进行开发。
  2. 数据绑定：Vue使用双大括号{{}}进行文本插值，而小程序使用双大括号进行内容绑定，但样式和属性绑定使用不同的语法。
  3. 显示与隐藏元素：Vue使用v-if和v-show控制元素的显示和隐藏，而小程序使用wx:if和hidden属性。
  4. 事件处理：Vue中可以使用v-on:event或@event绑定事件，而小程序使用bind或catch前缀加上事件名来绑定事件。
  5. 数据双向绑定：Vue支持数据双向绑定，而小程序则需要手动获取表单值并更新到data中。
  6. 组件化：两者都支持组件化开发，但组件的使用方式和语法有所不同。
  7. 路由与页面跳转：Vue使用路由进行页面跳转和导航，而小程序使用wx.navigateTo、wx.redirectTo等API进行页面跳转。
  8. 构建与发布：Vue项目需要构建打包后才能发布到服务器上，而小程序则直接上传代码包到微信开放平台进行审核和发布。
