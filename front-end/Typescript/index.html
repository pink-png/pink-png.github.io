TypeScript 知识点概览

一、TypeScript是什么，为什么要使用它？

产生背景

TypeScript是JavaScript的一个超集，添加了静态类型、接口、枚举等特性，解决了JavaScript在大型项目中类型不明确、难以维护等问题。

为何使用

  • 类型安全：提供静态类型系统，编译时捕获类型错误。
  • 工具支持：支持强大的自动补全、重构等编辑器功能。
  • 可维护性：提高代码的可读性和可理解性。
  • 可扩展性：支持大型项目，提供面向对象编程特性。

二、TS与JS的区别

特性 TypeScript JavaScript
类型系统 静态类型 动态类型
面向对象 支持类和接口 基于原型的继承
编译时检查 是 否（运行时检查）
IDE 支持 强大 有限
浏览器兼容性 需要编译为 JS 原生支持

四、基本类型

类型 描述 示例
number 任意数字 let x: number = 42

五、编译选项（tsconfig.json）

tsconfig.json 基本知识点

配置项 描述 示例
include 指定哪些文件或目录需要被编译 ["src/**/*"]（表示编译src目录及其子目录下的所有文件）
exclude 指定哪些文件或目录不需要被编译 ["node_modules", "**/*.spec.ts"]（排除node_modules目录和所有以.spec.ts结尾的文件）
extends 继承另一个tsconfig.json文件的配置 "./configs/base"（继承./configs/base目录下的tsconfig.json配置）
files 直接指定需要编译的文件列表 ["src/main.ts", "src/utils.ts"]（仅编译这两个文件）
compilerOptions 编译选项的集合，用于定制TypeScript编译器的行为 详见下方

compilerOptions 示例

子选项 描述 示例
target 指定编译输出的ECMAScript版本 "ES2015" 或 "ESNext"
module 指定模块化的规范 "commonjs" 或 "esnext"
lib 指定要包含在内的全局库文件列表 ["dom", "esnext"]
strict 启用所有严格的类型检查选项 true 或 false
esModuleInterop 允许默认导入与CommonJS模块互操作 true
jsx 指定JSX的编译模式 "react" 或 "react-jsx"
sourceMap 生成源映射文件，便于调试 true 或 false
outDir 输出目录，编译后的文件将放置在这里 "./dist"

请注意，compilerOptions中的子选项非常丰富，这里只列举了一些常见的选项。在实际使用中，可以根据项目需求进行选择和配置。

七、面向对象

面向对象编程（OOP, Object-Oriented Programming）是一种编程范式或方法，它将现实世界中的事物抽象为对象，并通过类和对象之间的交互来实现程序的逻辑。在TypeScript中，面向对象编程的核心概念包括类、接口、继承、封装和多态。

八、类（class）

（在TypeScript中，类是一种面向对象编程的构造，用于创建具有相同属性和方法的对象。

构造函数
构造函数是一种特殊的方法，当创建和初始化一个由类生成的对象时会被调用。

class Greeter {  
    greeting: string;  
    constructor(message: string) {  
        this.greeting = message;  
    }  
    greet() {  
        return "Hello, " + this.greeting;  
    }  
}  
  
let greeter = new Greeter("world");

继承
一个类可以继承另一个类的属性和方法，通过extends关键字实现。

class Animal {  
    name: string;  
    constructor(theName: string) { this.name = theName; }  
    move(distanceInMeters: number = 0) {  
        console.log(`${this.name} moved ${distanceInMeters}m.`);  
    }  
}  
  
class Dog extends Animal {  
    bark() {  
        console.log('Woof! Woof!');  
    }  
}  
  
let d = new Dog("Mitzie");  
d.bark();  
d.move(10);  
d.name = "Happy Dog";

九、接口

接口在TypeScript中是一个非常重要的概念，它定义了一个对象的结构，但不包含任何实现细节。

接口定义

interface LabelledValue {  
    label: string;  
      
    // 可选属性  
    value?: number;  
}  
  
function printLabel(labelledObj: LabelledValue) {  
    console.log(labelledObj.label);  
}  
  
let myObj: LabelledValue = {label: "size 10", value: 42};  
printLabel(myObj);

接口和类的区别

  • 接口只能定义方法签名和属性签名。
  • 类可以实现多个接口。
  • 类包含属性、方法和构造函数的具体实现。

十四、属性的封装

  • public：公有属性，可在类内部和外部访问。
  • private：私有属性，仅可在类内部访问。
  • protected：保护属性，在类内部和子类内部可访问。
  • getter/setter：用于获取和设置属性的值。
    在TypeScript中，可以使用getter和setter来控制对对象属性的访问和修改。
  class Employee {  
    private _fullName: string;  
  
    get fullName(): string {  
        return this._fullName;  
    }  
  
    set fullName(newName: string) {  
        this._fullName = newName;  
    }  

}  
  
let employee = new Employee();  
employee.fullName = "Bob Smith";  
console.log(employee.fullName);
